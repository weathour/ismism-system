# Phase 0：语料粒度与调用边界

## 1. 默认粒度

- 一个 `split_md` 文件 = 一个最小分析单元。
- 该单元已经绑定：
  - 一个目录编号 `toc_id`
  - 一个标题 `title`
  - 一段页码区间 `page_start -> next_page_start`
  - 一条父级路径 `path_ids` / `path_titles`

## 2. 何时只读当前单元

- 定义一个局部概念时
- 提取某个理论态的代表性表述时
- 比较同级兄弟节点差异时

## 3. 何时向上扩展

- 当前单元明显是“复习课”或“承前总结”
- 当前单元大量使用“这个一”“这个二”“前面说过”之类指代
- 当前单元没有给出自身矩阵位置的完整定义

## 4. 何时向下扩展

- 当前单元是总论、纲领、总标题
- 当前单元提出一个分类框架，但未展开所有分支
- 当前单元需要具体四级条目来补充理论变体

## 5. 何时横向扩展

- 需要建立同一父级下的四种演化状态
- 需要区分作者的褒贬、批判对象、机制差异
- 需要归纳“同场域不同本体论”或“同本体论不同认识论”的差别

## 6. 边界约束

- 不把不同 `toc_id` 的内容混写成一个假定定义。
- 不用外部哲学常识补齐缺口，所有定义必须回到原文链条。
- 若原文没有给出四元矩阵中的确切映射，必须显式标记为“待进一步抽样验证”。

## 7. 对后续矩阵工程的意义

- Phase 1 将在此粒度上建立概念库。
- Phase 2 将在此粒度上追踪跨节点跃迁与组合机制。
- Phase 3 将从已归档节点中抽取现实分析锚点。
- Phase 4 将把前述结果压缩成模块化 Skill 套件。
