## 5. CSV 字段含义

- `row_id`：当前条目的唯一行号
- `toc_id`：目录编号，如 `1-1-1-4`
- `tree_level` / `toc_depth`：目录层级
- `title`：当前条目标题
- `page_start` / `next_page_start`：起始页与下一条目的起始页
- `parent_row_id` / `parent_toc_id`：父级定位键
- `path_titles` / `path_ids`：从根到当前条目的完整路径
- `split_pdf_relpath` / `split_md_relpath`：拆分后 PDF / Markdown 的相对路径
- `split_pdf_exists` / `split_md_exists`：文件存在状态

## 6. 最小分析单元

- 默认最小单元是 `split_md/` 下与单条目录对应的一个 Markdown 文件。
- 每个单元都已经和 `toc_id`、父子关系、页码范围绑定。
- 后续所有概念提取、矩阵归类、跃迁分析，优先从 `split_md_relpath` 打开，不重复回到原始 PDF。

## 7. 编号稳定规则

- 后续新增分析文件尽量沿用 `toc_id` 编号体系。
- 引用原文时优先写成：`toc_id + title + split_md_relpath`
- 当需要聚合多个条目时，优先以共同父级 `toc_id` 建立专题文件，而不是重新发明命名规则。

## 8. 当前阶段结论

- Phase 0 的“目录地图”已经可直接作为后续 Phase 1-4 的统一导航层。
- 后续检索应以 `目录索引_结构化.csv` 为入口，以 `split_md/` 为正文层，以 `parent_row_id` / `toc_id` 为上下文扩展机制。
