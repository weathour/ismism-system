# Phase 0：机器检索协议

## 1. 检索总原则

- 先查 `目录索引_结构化.csv`，再开 `split_md/` 正文。
- 除非需要核对提取错误，不直接回到原始 PDF。
- 单次分析先锁定一个 `toc_id`，再决定是否向父级或子级扩展。

## 2. 标准检索流程

### 路径 A：已知目录编号

1. 用 `toc_id` 精确匹配 CSV。
2. 读取该行的 `split_md_relpath`。
3. 打开对应 Markdown，作为当前分析主文本。
4. 如语义不足，读取：
   - 父级：`parent_row_id` / `parent_toc_id`
   - 子级：所有 `toc_id` 以前缀 `当前 toc_id-` 开头的条目

### 路径 B：已知概念或标题关键词

1. 在 CSV 的 `title`、`path_titles` 中检索关键词。
2. 若结果过多，优先保留：
   - 标题精确命中
   - 四级节点
   - 位于同一父级链条的结果
3. 逐条读取 `split_md_relpath`，做概念去重与边界比对。

### 路径 C：已知理论层级但未知具体条目

1. 先锁定一级主部（`1`、`2`、`3`、`4`）。
2. 再按 `toc_id` 深度向下枚举。
3. 先读三级条目建立局部结构，再抽取对应四级细目。

## 3. 上下文扩展规则

- **向上扩展**：当某条目直接使用省略语、代词、复习口吻或默认前文定义时，读取父级条目。
- **向下扩展**：当某条目只提供总纲而缺少具体理论状态时，读取它的所有子条目。
- **横向比对**：当需要区分同层不同理论态时，读取同一父级下的兄弟节点。

## 4. 命名与存档规则

- 新文件优先放在 `Zhuyi_Matrix_Engine/` 下对应 Phase 文件夹。
- 文件名尽量体现用途，而不是重命名原始章节。
- 文件正文内部引用原文时，统一记录：
  - `toc_id`
  - `title`
  - `split_md_relpath`

## 5. 推荐检索键

- 结构键：`toc_id`、`parent_toc_id`、`tree_level`
- 语义键：`title`、`path_titles`
- 定位键：`page_start`、`split_md_relpath`
- 完整性键：`split_md_exists`

## 6. 为 Phase 1 预留的入口

- 与四元矩阵直接相关的核心词目前已确认至少包括：
  - `场域`
  - `本体论`
  - `认识论`
  - `目的论`
- 后续 Phase 1 将以这些关键词为第一轮抽样入口，再回接其所在 `toc_id` 链条。
